1. Introduction

This document describes CacheWatch, a watchdog/daemon that runs in the background and collects Last Level Cache hardware counters.

1.1. Compilation

Check the source code for more details. 

1: make

1.2. Testing

Check the makefile for more details. 

1: make test

The tests consist of running cachewatch using the four benchmarks decribed below on the same core (core 1).

  • Array reduction:
1: double r = 0.0;
2: 
3: for (unsigned long long i = 0; i < n; i++)
4:   r += a[i];
5: 
6: return r;
  • Dot product:
1: double d = 0.0;
2: 
3: for (unsigned long long i = 0; i < n; i++)
4:   d += a[i] * b[i];
5: 
6: return d;
  • GEMM :

    • IJK version: 

      1: for (unsigned long long i = 0; i < n; i++)
2:   for (unsigned long long j = 0; j < n; j++)
3:     for (unsigned long long k = 0; k < n; k++)
4:        c[i][j] += a[i][k] * b[k][j];

    • IKJ version: 

      1: for (unsigned long long i = 0; i < n; i++)
2:   for (unsigned long long j = 0; j < n; j++)
3:     {
4:       double _a_ = a[i][k];
5: 
6:       for (unsigned long long k = 0; k < n; k++)
7:         c[i][j] += _a_ * b[k][j];
8:     }

2. Hands on

2.1. Starting the watchdog

1: ./cachewatch --start CORE_NUM

If no core number is provided, the tool uses core 0 by default.

The watchdog process creates a pid (cachewatch.pid) file in the calling directory. This pid file is used to retrieve the PID of the watchdog in order for the tool to signal the end of the collection.

2.2. Stopping the watchdog

1: ./cachewatch --stop

Once the watchdog receives the stop signal, it dumps the collected counter values into an output file (cachewatch.out). If an output file already exists, it will be renamed (cachewatch.out.old).

3. Hardware Counters

3.1. Last Level Cache

CacheWatch uses the perf_event_open system call to set two Last Level Cache counters: PERF_COUNT_HW_CACHE_REFERENCES, PERF_COUNT_HW_CACHE_MISSES.
